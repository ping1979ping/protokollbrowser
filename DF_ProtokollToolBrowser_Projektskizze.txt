# DOCUframe Mobile Protokoll-App - Konzeption & JSON-Schnittstelle

> **Zweck dieses Dokuments:** KI-gestuetzte Weiterentwicklung in VS Code (Copilot, ChatGPT-Extension o.ae.).
> Alle Abschnitte sind so strukturiert, dass sie als Kontext fuer Code-Generierung, Schema-Verfeinerung und UI-Spezifikation genutzt werden koennen.

---

## 1. Zielsetzung

Entwicklung einer **offlinefaehigen, browserbasierten Protokoll-App (PWA)** fuer Baustellen-/Jour-fixe-Protokolle mit bidirektionaler Anbindung an **DOCUframe**.

**Kernfunktionen:**

- Protokollpunkte (aus DOCUframe exportiert) auf mobilen Geraeten im Browser anzeigen
- Checklistenfunktion: Punkte abhaken, Status aendern, Termine/Verantwortliche pflegen
- Neue Protokollelemente erfassen (inkl. Schnellerfassung "Mangel", "Restarbeit", "Info")
- Fotos (Kamera oder Galerie) an Protokollelemente anhaengen
- Standort (GPS) pro Eintrag erfassen - als Koordinaten + lesbarer Text
- Freitext oder Transkript per Device-Spracherkennung (Web Speech API) - **kein Audio speichern**
- **Vollstaendig offline** nutzbar (Service Worker + IndexedDB)
- Export als ZIP (JSON + Fotos) -> Import in DOCUframe via DOCUcontrol-Makro

---

## 2. Workflow-Uebersicht

```
DOCUframe (Desktop)
      |
      |  DOCUcontrol-Makro: Export
      v
  Protokolle.json
      |
      |  Datei-Upload im Browser
      v
+-----------------------------+
|   Mobile Protokoll-App      |  <- iPhone / iPad / Android / Windows-Tablet
|   (PWA, offline-faehig)     |
|                             |
|  - Elemente anzeigen        |
|  - Status setzen            |
|  - Fotos aufnehmen          |
|  - GPS erfassen             |
|  - Sprache -> Text          |
|  - Neue Elemente anlegen    |
+-----------------------------+
      |
      |  ZIP-Export (protocol_export.json + photos/)
      v
DOCUframe (Desktop)
      |
      |  DOCUcontrol-Makro: Import
      v
  Neues Protokoll N+1 angelegt          <- Nummer = max(vorhandene Nummern) + 1
  Elemente aktualisiert / neu erstellt
  Fotos: manuell im Projektstammverzeichnis ablegen
         (Pfad wird im JSON mitgeliefert, kein Auto-Import)
```

## 3. DOCUframe Datenmodell (Ist-Zustand)

### 3.1 GSDObjekt - Basisfelder (alle Objekte)

| Feldname      | Typ    | Bedeutung                        |
|---------------|--------|----------------------------------|
| `ExchangeID`  | STRING | Technische Austausch-ID (UUID)   |
| `CreateTime`  | TIME   | Erstellungszeitpunkt             |
| `ModifyTime`  | TIME   | Letzter Aenderungszeitpunkt      |

> **Wichtig:** `ExchangeID` ist **nicht** direkt als Feld zugaenglich. Stattdessen wird die OID via
> `Objekt.GetOID()` ausgelesen und als stabiler Primaerschluessel fuer den JSON-Roundtrip verwendet.
>
> ```pascal
> STRING OidStr;
> OidStr = Elem.GetOID();   // liefert z.B. "12345678-ABCD-..."
> ElemJSon.GetMember( "Id", TRUE ).SetString( OidStr );
> ```
>
> Beim Re-Import wird per `ObjektSet.FindOID( OidStr )` das Originalobjekt wiedergefunden.

## 4. Status-Mapping `_PINGProtokollelement._Status`

Quelle: DOCUframe Anzeigeparameter `_PINGProtokollelement.Status`

| Wert | Text            | Kategorie  | Hintergrund | Bedeutung / Verwendung                        |
|------|-----------------|------------|-------------|-----------------------------------------------|
| `0`  | neu             | -          | FFFFFF      | Neu erfasst, noch nicht bearbeitet            |
| `10` | offen           | -          | FFFFFF      | In Bearbeitung / offen                        |
| `19` | freigegeben     | -          | FFFFFF      | Freigegeben zur Ausfuehrung                   |
| `20` | erledigt        | -          | BBF2CD      | Abgeschlossen (gruener Hintergrund)           |
| `21` | uebertragen     | -          | FFFFFF      | In Folgeprotokoll uebertragen                 |
| `11` | offen (M)       | **Mangel** | FFFFFF      | Mangel - offen                                |
| `25` | beseitigt (M)   | **Mangel** | BBF2CD      | Mangel - beseitigt (gruener Hintergrund)      |
| `17` | erledigt (Info) | -          | FFFFFF      | Info-Punkt - erledigt                         |
| `16` | Restarbeit (R)  | **Mangel** | FFFFFF      | Restarbeit - offen (Versteckt: Ja)            |
| `26` | erledigt (R)    | **Mangel** | BBF2CD      | Restarbeit - erledigt (Versteckt: Ja)         |

### 4.1 Status-Gruppen fuer die Mobile App

```
Normale Punkte:
  0  -> neu
  10 -> offen
  19 -> freigegeben
  20 -> erledigt       (Abschluss-Status, gruen)
  21 -> uebertragen    (in Folgeprotokoll)

Maengel (Kategorie = "Mangel"):
  11 -> offen (M)
  25 -> beseitigt (M)  (Abschluss-Status, gruen)

Restarbeiten (Kategorie = "Mangel", versteckt in DF):
  16 -> Restarbeit (R)
  26 -> erledigt (R)   (Abschluss-Status, gruen)
  --> In v1 der App NICHT anzeigen/erfassen

Info-Punkte:
  17 -> erledigt (Info)
```

### 4.2 Erlaubte Status-Uebergaenge (Mobile App)

```
neu (0)
  +-> offen (10)
        +-> freigegeben (19)
        |     +-> erledigt (20)
        +-> erledigt (20)
              +-> uebertragen (21)  [nur DOCUframe]

offen (M) (11)
  +-> beseitigt (M) (25)

Restarbeit (R) (16)  [v1: nicht in App]
  +-> erledigt (R) (26)
```

### 4.3 Status-Schnellauswahl in der App (UI-Mapping)

| App-Button              | Setzt Status | Kategorie | Farbe  |
|-------------------------|-------------|-----------|--------|
| "Neu"                   | 0           | -         | grau   |
| "Offen"                 | 10          | -         | gelb   |
| "Freigegeben"           | 19          | -         | blau   |
| "Erledigt"              | 20          | -         | gruen  |
| "Mangel - offen"        | 11          | Mangel    | rot    |
| "Mangel - beseitigt"    | 25          | Mangel    | gruen  |

## 7. DOCUcontrol - Export-Makro (DOCUframe -> Mobile)

```pascal
// DOCUcontrol-Makro: Export Protokollgruppe + Protokoll + Elemente -> JSON
// WICHTIG: GetOID() statt ExchangeID verwenden!
// ProjektStammverzeichnis wird mitexportiert.
// Teilnehmer/Verteiler werden mit OID, Nummer und Name exportiert.

HJSON JSon;
HJSON GroupJSon;

## 8. DOCUcontrol - Import-Makro (Mobile -> DOCUframe)

```pascal
// DOCUcontrol-Makro: Import protocol_export.json -> neues Folgeprotokoll anlegen
// - Protokollnummer N+1 wird automatisch berechnet (max + 1)
// - Verantwortlicher wird via OID (FindOID) gesetzt
// - Fotos: kein Auto-Import, ZielPfad wird in Bemerkung vermerkt
// - Nur ein Mitarbeiter gleichzeitig (kein Konflikt-Handling noetig)

HJSON JSon;
HJSON GroupJSon;
HJSON ProtoJSon;
HJSON ElemArray;
HJSON ElemJSon;
HJSON MobileJSon;
HFILE File;
STRING JSonTxt;
INT GroupCount, i, j, ElemCount;
INT MaxNummer, NeueNummer;
STRING GroupId, DfElementId, Aktion, GeoText, Transkript;
STRING VerantwOid;

_PINGProtokollgruppeSet ProtGrpSet;
_PINGProtokollgruppe ProtGrp;
_PINGProtokollSet ProtSet;
_PINGProtokoll Prot, ProtAlt;
_PINGProtokollelementSet ElemSet;
_PINGProtokollelement Elem;
AdresseNrSet AdrSet;
AdresseNr Adr;

JSon.Construct();

File.Open( "C:\\Temp\\protocol_export.json", FILE_MODE_READ );
FileReadString( File, JSonTxt, FileGetLength( File ), TRUE );
File.Close();
JSon.SetText( JSonTxt );

GroupCount = JSon.GetElementCount();
IF( GroupCount > 0 )

  FOR( i = 1; i <= GroupCount; i++ )
    GroupJSon = JSon.GetElement( i );
    GroupId = GroupJSon.GetMember( "ProtokollgruppeId", TRUE ).GetString();

    // Protokollgruppe via OID suchen
    ProtGrpSet.Construct( "_PINGProtokollgruppe" );
    IF( ProtGrpSet.FindOID( GroupId ) )
      ProtGrpSet.Get( ProtGrp, 0, CURRENT );
    ELSE
      // Fehlerbehandlung: Gruppe nicht gefunden
    ENDIF

    // Maximale Protokollnummer ermitteln -> N+1
    MaxNummer = 0;
    ProtSet = ProtGrp._Protokolle;
    IF( ProtSet.Get( ProtAlt, 1, START ) )
      REPEAT
        IF( ProtAlt._Nummer > MaxNummer )
          MaxNummer = ProtAlt._Nummer;
        ENDIF
      UNTIL( !ProtSet.Get( ProtAlt, 1, CURRENT ) )
    ENDIF
    NeueNummer = MaxNummer + 1;

    // Neues Folgeprotokoll anlegen
    ProtoJSon = GroupJSon.GetMember( "ProtokollMeta", TRUE );
    Prot.Create( "_PINGProtokoll" );
    Prot._Protokollgruppe = ProtGrp;
    Prot._Name = ProtoJSon.GetMember( "Name", TRUE ).GetString();
    Prot._Nummer = NeueNummer;
    Prot._Datum = ProtoJSon.GetMember( "Datum", TRUE ).GetString();
    Prot._Ort = ProtoJSon.GetMember( "Ort", TRUE ).GetString();
    Prot._Autor = ProtoJSon.GetMember( "Autor", TRUE ).GetString();
    Prot._Vorbemerkung = ProtoJSon.GetMember( "Vorbemerkung", TRUE ).GetString();
    Prot._Nachbemerkung = ProtoJSon.GetMember( "Nachbemerkung", TRUE ).GetString();
    Prot._erstellt = TRUE;
    Prot.Store();

    // Elemente verarbeiten
    ElemArray = GroupJSon.GetMember( "Elemente", TRUE );
    ElemCount = ElemArray.GetElementCount();

    FOR( j = 1; j <= ElemCount; j++ )
      ElemJSon = ElemArray.GetElement( j );
      Aktion = ElemJSon.GetMember( "Aktion", TRUE ).GetString();
      MobileJSon = ElemJSon.GetMember( "MobileDaten", TRUE );
      GeoText = MobileJSon.GetMember( "GeoText", TRUE ).GetString();
      Transkript = MobileJSon.GetMember( "Transkript", TRUE ).GetString();
      VerantwOid = ElemJSon.GetMember( "VerantwortlicherOidNeu", TRUE ).GetString();

      IF( Aktion = "UPDATE" )
        DfElementId = ElemJSon.GetMember( "DfElementId", TRUE ).GetString();
        ElemSet.Construct( "_PINGProtokollelement" );
        IF( ElemSet.FindOID( DfElementId ) )
          ElemSet.Get( Elem, 0, CURRENT );
          IF( Elem.Lock( WRITE_NOWRITE, FLAT ) )
            IF( Elem.Refresh( FLAT ) )
              Elem._Status = ElemJSon.GetMember( "StatusNeu", TRUE ).GetNumber();
              Elem._Termin = ElemJSon.GetMember( "TerminNeu", TRUE ).GetString();
              Elem._Bemerkung = ElemJSon.GetMember( "BemerkungNeu", TRUE ).GetString();
              IF( GeoText <> "" )
                Elem._Bemerkung = Elem._Bemerkung + "\r\n[GPS] " + GeoText;
              ENDIF
              IF( Transkript <> "" )
                Elem._Bemerkung = Elem._Bemerkung + "\r\n[Transkript] " + Transkript;
              ENDIF
              // Verantwortlicher via OID setzen
              IF( VerantwOid <> "" )
                AdrSet.Construct( "AdresseNr" );
                IF( AdrSet.FindOID( VerantwOid ) )
                  AdrSet.Get( Adr, 0, CURRENT );
                  Elem._Verantwortlicher = Adr;
                ENDIF
              ENDIF
              // Fotos: ZielPfad in Bemerkung vermerken (kein Auto-Import)
              // TODO: MobileDaten.Fotos[].ZielPfad auslesen und in Bemerkung anhaengen
              Elem.Store( FLAT );
            ENDIF
            Elem.Unlock( WRITE_NOWRITE, FLAT );
          ENDIF
        ENDIF

      ELSEIF( Aktion = "CREATE" )
        Elem.Create( "_PINGProtokollelement" );
        Elem._Protokoll = Prot;
        Elem._Position = ElemJSon.GetMember( "Position", TRUE ).GetString();
        Elem._Positionstitel = ElemJSon.GetMember( "Positionstitel", TRUE ).GetString();
        Elem._Positionstext = ElemJSon.GetMember( "Positionstext", TRUE ).GetString();
        Elem._Thema = ElemJSon.GetMember( "Thema", TRUE ).GetString();
        Elem._Status = ElemJSon.GetMember( "Status", TRUE ).GetNumber();
        Elem._Termin = ElemJSon.GetMember( "Termin", TRUE ).GetString();
        Elem._Bemerkung = ElemJSon.GetMember( "BemerkungNeu", TRUE ).GetString();
        IF( GeoText <> "" )
          Elem._Bemerkung = Elem._Bemerkung + "\r\n[GPS] " + GeoText;
        ENDIF
        IF( Transkript <> "" )
          Elem._Bemerkung = Elem._Bemerkung + "\r\n[Transkript] " + Transkript;
        ENDIF
        // Verantwortlicher via OID setzen
        IF( VerantwOid <> "" )
          AdrSet.Construct( "AdresseNr" );
          IF( AdrSet.FindOID( VerantwOid ) )
            AdrSet.Get( Adr, 0, CURRENT );
            Elem._Verantwortlicher = Adr;
          ENDIF
        ENDIF
        Elem.Store();
      ENDIF

    NEXT
  NEXT
ENDIF

JSon.Destruct();
```

### 9.5 GPS-Erfassung (TypeScript)

```typescript
async function getPosition(): Promise<{
  lat: number;
  lon: number;
  accuracy: number;
  text: string;
}> {
  return new Promise((resolve, reject) => {
    navigator.geolocation.getCurrentPosition(
      (pos) => {
        const lat = pos.coords.latitude;
        const lon = pos.coords.longitude;
        const accuracy = Math.round(pos.coords.accuracy);
        const text =
          "Standort: " +
          lat.toFixed(7) +
          ", " +
          lon.toFixed(7) +
          " (Genauigkeit " +
          accuracy +
          " m)";
        resolve({ lat, lon, accuracy, text });
      },
      (err) => reject(err),
      { enableHighAccuracy: true, timeout: 10000, maximumAge: 0 }
    );
  });
}

function startSpeechRecognition(onResult: (text: string) => void): void {
  const SpeechRecognition =
    (window as any).SpeechRecognition ||
    (window as any).webkitSpeechRecognition;
  if (!SpeechRecognition) {
    alert("Spracherkennung wird von diesem Browser nicht unterstuetzt.");
    return;
  }
  const recognition = new SpeechRecognition();
  recognition.lang = "de-DE";
  recognition.interimResults = false;
  recognition.maxAlternatives = 1;
  recognition.onresult = (event: any) => {
    const transcript = event.results[0][0].transcript;
    onResult(transcript);
  };
  recognition.start();

### 9.7 ZIP-Export (TypeScript)

```typescript
import JSZip from "jszip";

async function exportZip(
  exportJson: object,
  fotos: { fileName: string; blob: Blob }[]
): Promise<void> {
  const zip = new JSZip();
  zip.file("protocol_export.json", JSON.stringify(exportJson, null, 2));
  const photosFolder = zip.folder("photos")!;
  for (const foto of fotos) {
    photosFolder.file(foto.fileName, foto.blob);
  }
  const content = await zip.generateAsync({ type: "blob" });
  const url = URL.createObjectURL(content);
  const a = document.createElement("a");
  const ts = new Date()
    .toISOString()
    .replace(/[:.]/g, "-")
    .slice(0, 19);
  a.href = url;
  a.download = "protocol_export_" + ts + ".zip";
  a.click();
  URL.revokeObjectURL(url);
}
```

## 10. Geklärte Designfragen (Zusammenfassung)

| # | Frage | Antwort / Entscheidung |
|---|-------|------------------------|
| 1 | ExchangeID-Verfügbarkeit | `ExchangeID` nicht direkt zugänglich. Stattdessen `Objekt.GetOID()` verwenden. Re-Import via `ObjektSet.FindOID(OidStr)`. |
| 2 | Foto-Import in DF | Kein automatischer Import als Dokument-Objekt. Fotos werden im Projektstammverzeichnis abgelegt. Pfad wird im JSON mitgeliefert. |
| 3 | Verantwortlicher-Lookup | OID der Adresse wird exportiert. In der App Auswahl aus Teilnehmern/Verteilern. Re-Import via `FindOID`. |
| 4 | Protokollnummer N+1 | Wird beim Re-Import in DOCUframe automatisch berechnet (`max(Nummer) + 1`). |
| 5 | Multi-User | Nur ein Mitarbeiter gleichzeitig pro Protokoll. Keine Konfliktlösung (Locking) in v1 nötig. |
| 6 | Versteckte Status (16, 26) | Restarbeiten (Status 16/26) sind für Mängellisten gedacht. In v1 der App werden diese **nicht** angezeigt oder erfasst. |
| 7 | Kategorie-Feld | `_PINGProtokollgruppe._Kategorie` ist für die App-Logik/Filterung aktuell nicht relevant. |
